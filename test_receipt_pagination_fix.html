Receipt Pagination Fix Test

This test verifies that the frontend now fetches all receipts instead of just the first page.

Issue:

  • Backend has 107 receipts total
  • Frontend was only showing 20 receipts (first page)
  • New receipts were being saved but not appearing in the list

Root Cause:

The frontend was calling receiptService.getReceipts() without pagination parameters, which returned only the default page size (20 items).

Fix Applied:

Updated all receiptService.getReceipts() calls to include { page_size: 200 } parameter to fetch all receipts.

Files Modified:

  • frontend/src/pages/ReceiptPageV2.jsx - 4 locations updated

Test Steps:

  1. Refresh the receipts page
  2. Upload a new receipt
  3. Verify the receipt count increases and the new receipt appears
  4. Check that all 107+ receipts are now visible

Expected Result:

The receipt list should now show all available receipts (107+) and new uploads should immediately appear in the list.
